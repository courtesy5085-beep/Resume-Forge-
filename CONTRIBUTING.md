# ResumeForge AI - Contributing Guidelines

We love your input! We want to make contributing to ResumeForge AI as easy and transparent as possible.

## Development Process

We use GitHub to host code, to track issues and feature requests, as well as accept pull requests.

### Pull Requests

1. Fork the repo and create your branch from `main`
2. If you've added code that should be tested, add tests
3. Ensure the test suite passes
4. Make sure your code follows the style guide
5. Issue that pull request!

### Coding Style

- Use TypeScript for all new code
- Follow the ESLint configuration
- Run `pnpm format` before committing
- Write meaningful commit messages

### Testing

- Write tests for new features
- Ensure all tests pass: `pnpm test`
- Aim for >80% code coverage

### Documentation

- Update README.md with any new features
- Comment complex logic
- Keep documentation up-to-date

## Reporting Bugs

Report bugs using GitHub Issues. Include:
- Expected behavior
- Actual behavior
- Steps to reproduce
- Screenshots/logs if applicable

## Feature Requests

We welcome feature requests! Please:
- Use a clear, descriptive title
- Provide a clear description
- Include use cases and benefits

## Code Review

All submissions require review. We use GitHub pull request reviews.
